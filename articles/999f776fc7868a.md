## はじめに
VScodeのIntegrated browser（統合ブラウザ）機能を試してみました。
Github Copilotと組み合わせて結構便利に使えると感じたので、紹介していこうと思います。

:::message
Integrated browser機能は、この記事を書いている2026年3月22日時点ではExperimentalな機能なので、将来的に変更が入る可能性がある点はご注意ください。
:::

## バージョン
- VScode：1.112.0
- Github Copilot Chat：0.40.1
:::message
バージョンが違えば挙動や見た目が変わる可能性があります。
:::

## Integrated browserとは
> Integrated browserを使用すると、VScode内で直接Webページを開いて操作することができます。これを利用して、Webアプリケーションのプレビュー表示や認証フローのテストを行うほか、ページ上の要素を選択して、AIチャットのプロンプトにコンテキストとして追加することも可能です。

こちら公式ドキュメントの説明を日本語訳しただけですが、十分わかりやすい説明となっています。

https://code.visualstudio.com/docs/debugtest/integrated-browser
今回の記事で紹介する内容は、基本こちらの公式ドキュメントを見て、試してみたものとなります。Integrated browserがどういうものかや使い方について、一番正しい情報としてこちらのサイトを適宜確認いただければと思います。

自分はCursorユーザーではないですが、CursorにもBrowser機能（下記リンク）があり、それと同じような機能をVScodeも出してきたという印象です。
https://cursor.com/ja/docs/agent/tools/browser
:::message
まだCursorの方が機能的には少し充実しているようにも感じますが、十分VScodeも使えるくらい便利だと思いました。
ドキュメントを読むと、「サイドバーで強力なビジュアル編集ができる機能がある」という点はCursorにしかなさそうでしたが、それ以外は同じようなことができそうに感じています。
:::

## 基本的な使い方
### 画面の開き方
コマンドパレット（`Shift+Cmd+P`）から「`Browser: Open Integrated Browser`」コマンドを実行します。
![](/images/999f776fc7868a/1.png =500x)
あとは普通のブラウザと同じように利用することができるので、URL部分に確認したいアプリのURLを入力しましょう。下の画像ではとりあえずローカル起動したアプリのURLを入力していますが、公開されているアプリも当然Integrated browserで開くことができます。
![](/images/999f776fc7868a/2.png =500x)

### AIチャットに要素を伝える
![](/images/999f776fc7868a/3.png =500x)
赤枠のボタンをクリックすると要素を選択できるようになります。
![](/images/999f776fc7868a/4.png =500x)
修正したい箇所の要素をクリックすると
![](/images/999f776fc7868a/5.png =500x)
チャットにHTML、CSS、スクリーンショットなどの情報が付与され、ピンポイントに指示を与えることができます（messages.tsは自分が開いていただけでこの機能とは無関係となります）。
![](/images/999f776fc7868a/6.png =500x)
実際にこれだけの指示で、変えたい要素の色を青から緑に変更することに成功しました。変更したい箇所のコードを人が特定して伝えなくても、「画面上のここを直して欲しい」といった感じで、簡単に伝えられるのは非常に便利ですね。

含める情報は以下の設定などで多少変えられるようです。
- chat.sendElementsToChat.attachCSS
- chat.sendElementsToChat.attachImages

### AIチャットにブラウザのconsole出力を伝える
![](/images/999f776fc7868a/7.png =500x)
要素を伝えるボタンの右にあるボタンをクリックすると、ブラウザconsoleの出力情報をAIチャットに加えることができます。
consoleでエラーなど出ている場合に、わざわざコピペをする必要がなくなるので、これも便利ですね。
:::message
Console Logsや要素の情報などはcopilotのチャット部分でクリックすれば、実際にどのような情報がAIに渡されるのか詳細を確認できるので、思ったような挙動にならない場合は、詳細を確認してみるのが良いかと思いました。
:::

## より自由にエージェントにブラウザ操作させる（Browser tools for agents）
これまではユーザーがブラウザを見て、copilotへの指示出しを便利にする機能を紹介してきました。`Browser tools for agents`という機能を使えば、さらに自由にエージェントがブラウザから情報を取得したり、ブラウザ操作できるようです。
### どんなことが可能か
copilotからIntegrated browserの画面に対して、以下のようなことを外部MCPサーバーなしで行えるようです。
- エージェントはブラウザページのオープン
- URLへの移動
- ページコンテンツとコンソールエラーの読み取り
- スクリーンショット撮影
- 要素のクリック・テキスト入力・ホバー・ドラッグ
- ダイアログ処理
- Playwrightコードの実行

Playwright MCPを使ってこれらのことは可能でしたが、MCPなしでもできるのは便利だなと思いました。

### 使用するための条件
まず、この機能を使うためには、`workbench.browser.enableChatTools`の設定をtrueにする必要があります。
そして、エージェントがアクセスできるのは、エージェント自身がopenBrowserPageツールで開いたページか、ユーザーが「Share with Agent（エージェントと共有）」ボタンで明示的に共有したページのみのようです。

### 方法1：エージェント自身がopenBrowserPageツールで開いたページ
![](/images/999f776fc7868a/8.png)
*画面操作させてみた結果の例*

例えば、以下のように「統合ブラウザ」を使うように指示をすると新しくエージェントがIntegrated browserを開き、画面を操作してくれました。
```
アプリは起動しているので、統合ブラウザで「こんにちは」というスレッドを選択し、質問入力欄に「おやすみ」と入力して。送信はしなくて良いです。
```
同じチャットのセッションであれば、ブラウザ操作の権限を持った状態を維持できるらしく、続けてcopilotと会話して、さらに情報取得したり、ブラウザ操作させることが可能です。

### 方法2：Share with Agent（エージェントと共有）
こちらは既に開いているIntegrated browserの操作権限をcopilotのエージェントに与えるための機能になると思います。
![](/images/999f776fc7868a/9.png)
*公式ドキュメントのShare with Agentの画像*

公式ドキュメントは上画像の赤枠ボタンをクリックすることで、Sharing with Agentと表示され、この機能が使えると書かれていましたが、自分の画面ではボタンは存在するものの、クリックしても何も起きなかったので、まだ使えない機能かもしれません。
方法1では毎回新規のブラウザが立ち上がるため、既にブラウザを開いている場合は面倒なので、方法2が早く使えるようになって欲しいです。
